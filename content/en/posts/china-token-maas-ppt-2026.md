## 1. Executive Summary: Three Main Lines Determine the Winner of MaaS in 2026

![Three Main Lines: Token Scaling / Cost Curve Decline / Ecosystem Gateway Reordering](/tech-blog/images/maas-ppt/01-three-main-lines.svg)

**Line One: Token Scaling** — Average daily call volume becomes the number one indicator of cloud vendors' AI commercialization. ByteDance disclosed that Doubao's average daily Token usage exceeded 120 trillion, while Alibaba disclosed that Bailian's Token consumption increased 6x in three months.

**Line Two: Cost Curve Decline** — Pricing moves from pay-as-you-go API to multi-tier structures including input/output Tokens, context length, Batch, caching, package plans, etc. Vendors use low-priced models to lock in developers.

**Line Three: Ecosystem Gateway Reordering** — Alibaba leans toward cloud and enterprise full-stack, ByteDance leans toward application traffic and content/Agent, Tencent leans toward enterprise connectivity, WeChat ecosystem, and TokenHub aggregation gateway.

> **Investment and Business Implication**: The core of MaaS is not the margin of a single model, but the combined return of "high-frequency Token traffic + low marginal inference cost + application ecosystem conversion."

Sources: Volcano Engine, Alibaba Cloud Bailian, Tencent Cloud TokenHub, public reports

## 2. Token Economics: Revenue Design Coupled with Inference Cost

![Token Economics Formula: Token = usage + pricing + compute](/tech-blog/images/maas-ppt/02-token-economics.svg)

### Revenue Side

- Priced separately by input/output Tokens, **output is usually more expensive**
- Long context, reasoning mode, multimodal, video generation create price gradients
- Token Plans, annual/monthly packages, and enterprise committed consumption convert volatile calls into predictable revenue

### Cost Side

- GPU/domestic AI chips, HBM, networking, and electricity determine the base inference cost
- Batch invocation, context caching, distillation, and small model routing reduce unit cost
- Agent long chains amplify Token consumption but also increase stickiness and per-customer value

### Key Variable × Business Implication × Management Focus

| Key Variable | Business Implication | Management Focus |
|---|---|---|
| **Price per million Tokens** | Determines developer entry barrier and replacement cost | Price band, packages, free quota |
| **Unit inference cost** | Determines MaaS gross margin elasticity | Model compression, scheduling, caching, chip supply |
| **Token retention rate** | Determines whether it shifts from trial to production environment | SLA, toolchain, data security, Agent framework |

> **One-sentence Formula**: `Token = usage + pricing + compute`

## 3. MaaS Market: Upgrading from Model API to AI Operating System

![MaaS Three-layer Stack: Model Layer / Platform Layer / Application Layer](/tech-blog/images/maas-ppt/03-maas-stack.svg)

| Layer | Capabilities | Value Capture | Competitive Focus |
|---|---|---|---|
| **Model Layer** | General, reasoning, code, multimodal, video, voice | Charged per Token/task | Performance, cost, context, open-source/closed-source strategy |
| **Platform Layer** | Model marketplace, API gateway, evaluation, monitoring, data governance | Enterprise contracts, packages, cloud resource linkage | Developer experience, SLA, compliance and private deployment |
| **Application Layer** | Agent, office, customer service, marketing, search, code generation | Subscription, value-added, advertising, e-commerce conversion | Scenario closure, user entry points, workflow stickiness |

> **2026 Key Judgment**: Competition is no longer about "who has the model," but who can embed the model into enterprise workflows and consumer entry points, and settle high-frequency calls into predictable Token revenue.

## 4. ByteDance Volcano Engine: Driving the Token Flywheel with Application Traffic

![ByteDance Volcano Engine: Doubao 120 trillion + Entry Matrix](/tech-blog/images/maas-ppt/04-bytedance.svg)

**Core Number: Doubao Average Daily Token Usage Exceeds 120 Trillion**

## 6. Tencent Hunyuan + TokenHub: Betting on the Aggregation Gateway

![Tencent TokenHub: 50-80% Discount + Aggregation Gateway](/tech-blog/images/maas-ppt/06-tencent.svg)

### Platform Entry

**Tencent Cloud TokenHub is positioned as a unified large model service gateway**, integrating Hunyuan and introducing third-party models, covering scenarios such as conversation, reasoning, code, vision, image, and video.

**Package Discount: 50%-80%** — TokenHub product page emphasizes that packages are 50%-80% cheaper than pay-as-you-go API pricing, using prepaid and usage packs to reduce developers' perceived marginal cost.

### Model Iteration

Hunyuan Hy3 preview has been open-sourced; public information shows input pricing around **1.2 CNY / million Tokens**, and Token Plans launched for Agent developers.

### SWOT

| Strengths | Constraints |
|---|---|
| WeChat/WeCom/Tencent Meeting/cloud and security ecosystem, aggregation platform facilitates enterprise multi-model governance | Needs to balance value capture between in-house model mindshare and third-party aggregation |

> **Observation Metrics**: TokenHub paid plan repurchase, Hunyuan open-source ecosystem, WeChat scenario call volume

Sources: Tencent Cloud TokenHub product page; Tencent Cloud Developer Community

## 8. Strategic Implications: Reshaping Cloud, Software, and Developer Product Design

![Impact Map: Cloud / Enterprise / Developer / Investor Quadrants](/tech-blog/images/maas-ppt/08-implications.svg)

### For Cloud Vendors

- MaaS will become the new front-end for cloud resources, bundling GPU, storage, networking, and data security behind it
- Price war is unavoidable; the key is to maintain cost advantages with caching, Batch, model routing, and in-house chips

### For Enterprise Customers

- Procurement criteria shift from model benchmark scores to **total cost of ownership, stability, data isolation, and workflow integration**
- Agent projects need to establish Token budgets, effectiveness attribution, and automatic downgrade strategies

### For Developers

- **Multi-model routing, prompt caching, context compression, result reuse** will become the fundamentals of application profitability

### For Investors

- Should track **Token consumption, paid retention, unit inference cost, AI cloud revenue share**, not just model release cadence

## 9. 2026-2027 Outlook: From Low-Cost Ubiquity to Efficiency Tiering

![2026-2027 Outlook: Trend Timeline](/tech-blog/images/maas-ppt/09-outlook.svg)

### Three Predictions

1. **Pricing continues to tier**: Flagship models rise in price or remain stable, Lite/Mini models carry the ubiquitous traffic
2. **Enterprise MaaS moves from pilot to production**: Agent, customer service, code, marketing content scale first
3. **Platforms shift from selling APIs to selling governance**: Model marketplace, auditing, routing, cost control become standard

### Recommendations

| Role | Recommendation |
|---|---|
| **Vendors** | Lock in high-frequency developers with Token Plans while publicly demonstrating SLA and cost optimization capabilities |
| **Enterprises** | Establish multi-vendor strategies, split model tiers by task to avoid single-model cost lock-in |
| **Developers** | Incorporate Token cost into product design, default to caching, compression, and model routing |
