## 背景：「Token / MW」で測る理由

AI コンピューティング競争において、**電力は究極の制約**です。どんなにチップが強力でも、最終的には「どれだけの電力コストで、どれだけの token を生成できるか」が問われます。

この指標は以下の要素を総合します：
- GPU の消費電力（TDP）
- GPU の計算能力（TFLOPS）
- GPU の数量（電力上限による制限）
- 実際の推論効率（メモリ帯域幅、インターコネクト帯域幅による影響）

## 理論演算能力：1 MW あたり何枚搭載できるか？

$$N_{cards} = \frac{1{,}000{,}000 \text{ W}}{TDP_{per\_card}}$$

| チップ | 1カードあたり消費電力 | 1 MW あたり搭載可能枚数 |
|---|---|---|
| H100 / H200 | 700W | 約 1,428 枚 |
| B100 | 700W | 約 1,428 枚 |
| B200 | 1,000W | 1,000 枚 |

---

## FP16 演算能力：1 MW あたりの総演算能力

$$P_{MW} = N_{cards} \times FP16_{per\_card}$$

| チップ | 1カードあたり FP16 | 1 MW あたり搭載枚数 | 1 MW あたり総演算能力 |
|---|---|---|---|
| H100 | 3,956 TFLOPS | 1,428 | **5.65 PFLOPS** |
| H200 | 3,956 TFLOPS | 1,428 | **5.65 PFLOPS** |
| B100 | 14,400 TFLOPS | 1,428 | **20.6 PFLOPS** |
| B200 | 20,000 TFLOPS | 1,000 | **20 PFLOPS** |

**結論**：B100 は Blackwell アーキテクチャにより、同一電力で H100 の **3.6 倍** の総演算能力を達成。

## Token 生成効率の推定

### 推定方法

FP16 演算能力 ≠ Token 生成量。実際の効率は以下に依存します：
1. **メモリ帯域幅**：データ供給速度を決定
2. **インターコネクト帯域幅**：マルチ GPU 連携効率を決定
3. **モデルサイズ**：大規模モデルはメモリ帯域幅がボトルネック
4. **バッチサイズ**：バッチ処理で利用率向上が可能

**推定基準**：Llama-70B モデルを例とする

| チップ | Token/s per カード（推定） | 参考 |
|---|---|---|
| H100 SXM | 2,500 – 4,000 tok/s | Llama-70B @ FP16 |
| H200 | 3,000 – 5,000 tok/s | 同上、HBM3e により向上 |
| B100 | 7,000 – 10,000 tok/s | Blackwell アーキテクチャによる向上 |
| B200 | 8,000 – 12,000 tok/s | NVLink 5.0 により向上 |

## Token / MWh 比較

$$\text{Token per MWh} = \text{Token/s per card} \times N_{cards} \times 3600$$

### 静的デプロイ（アイドルタイムなし）

| チップ | Token/s per カード | 1 MW あたりカード数 | **Token / MWh（静的）** |
|---|---|---|---|
| H100 | 3,500 | 1,428 | **17.9 T** |
| H200 | 4,200 | 1,428 | **21.6 T** |
| B100 | 8,500 | 1,428 | **43.7 T** |
| B200 | 10,000 | 1,000 | **36.0 T** |

> T = 兆（10¹²）

### 動的利用率（実際のデプロイ効率を考慮）

実際の推論では、GPU が 100% 稼働することはありません。ピーク利用率を考慮した場合：

| チップ | 静的 Token/MWh | 利用率 70% |
|---|---|---|
| H100 | 17.9 T | **12.5 T** |
| H200 | 21.6 T | **15.1 T** |
| B100 | 43.7 T | **30.6 T** |
| B200 | 36.0 T | **25.2 T** |

## 主要結論

### 1. B200 は現在 Token 効率が最も高い GPU

B200 の 1 MW あたり Token 効率は H100 の約 **2 倍**、総演算能力は H100 の **3.6 倍** です。

Blackwell アーキテクチャの主な利点：
- **NVLink 5.0**：1.8 TB/s 双方向インターコネクトにより、マルチ GPU 通信のボトルネックを完全に解消
- **192GB HBM3e**：1 カードで 70B+ モデルを搭載可能、バッチサイズが拡大
- **20 PFLOPS FP16**：単位消費電力あたりの演算能力が大幅に向上

### 2. H200 は現時点で最もバランスの取れた選択肢

H200 は H100 と同一の消費電力（700W）ながら、141GB HBM3e メモリを搭載：
- メモリ容量が 76% 向上
- メモリ帯域幅が 43% 向上
- Token 効率は H100 より **約 20%** 高い

既存の H100 クラスターを持つ企業にとって、H200 へのアップグレードは最も費用対効果の高い選択肢です。

### 3. インターコネクト帯域幅が中核的ボトルネック

大規模モデルの推論では、NVLink が生命線です：
- 70B モデルは 1 カードに収まらず、マルチ GPU 並列処理が必須
- HCCS/PCIe 帯域幅は NVLink よりはるかに低く、マルチ GPU 利用率が大幅に低下
- B200 の NVLink 5.0（1.8 TB/s）は H100 の NVLink 4.0（900 GB/s）より **2 倍**高速

### 4. 消費電力 vs 演算能力のトレードオフ

B200 の消費電力は H100 より 43% 高い（1000W vs 700W）ものの、総演算能力は H100 の 3.6 倍です。

これは以下を意味します：
- データセンターの冷却設計のアップグレードが必要
- 1 MW あたりのラック数は減少するが、総演算能力は大幅に向上
- **1 Token あたりの電力コストは実質 50% 以上低下**
