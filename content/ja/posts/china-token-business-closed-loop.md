## 一言でまとめると

> **中国におけるトークンビジネスのクローズドループは部分的に成立している——特定分野のトップモデル（動画／コーディング）ではすでにループが閉じており、汎用LLMは半クローズド状態。MaaSの販売パラダイムは「形」は学べても「神髄」は学べない。1/10の価格で回る仕組みの背後には、5つのメカニズム（長文コンテキスト税／出力インフレ／スケジューリング割引／予算内製化／コンプライアンスプレミアム）と、3つの中国独自の強み（国産GPU／電力補助金／オープンソースのフライホイール）が存在する。**

##  1、まず3つの問題を整理する

本稿では、[国内MaaS大戦](/tech-blog/posts/domestic-maas-token-wars-2026/)（「二つの戦線」について既述）や、[中国トークンの海外進出](/tech-blog/posts/china-token-going-global-2026/)（「OpenRouterのシェア61%」について既述）の内容は繰り返さない。**今回は以下の3つの新しい問いにのみ答える。**

1.  **ビジネスのクローズドループ** — トークン販売収入で、計算リソース＋研究開発＋販売コストをカバーできるのか？成功している企業はどこか？
2.  **好循環** — クローズドループが成立した後、事業は自己強化されるのか？（モデル向上 → 顧客増加 → データ還流 → さらなるモデル向上、という流れ）
3.  **価格が1/10でも事業が回る仕組み** — 中国のAPI価格は長期的に海外の10分の1以下だが、市場は崩壊していない。それを支える5つの基盤メカニズムとは何か？

##  2、クローズドループ／好循環の厳密な計算

### 2.1 トークン単価のコスト構造（推論側、2026年6月時点の逆算）

中国の主流GPUサーバー＋電力コストから逆算する：

|---|---|---|
| GPUサーバー調達 | ¥350,000/台 | H100 8基 / H200 8基 / 国産Aカード 8基 |
| サーバー減価償却 | 3年定額 | 月額 ¥9,720 |
| 電気代 | ¥0.65/kWh（税込・補助金込） | 上海/北京/深センIDC |
| 1台あたり最大消費電力 | 5.6 kW | GPU 8基 + CPU + 冷却 |
| 1台あたり月間電気代 | ¥2,624 | |
| 運用保守＋ラックスペース | ¥3,000/台/月 | |
| 帯域＋ストレージ | ¥1,500/台/月 | |
| 人件費按分 | ¥4,000/台/月 | |
| **1台あたり月間総コスト** | **¥20,844** | |
| 1台あたり月間推論トークン生産能力 | 500億トークン | 稼働率30%を想定 |
| **100万トークンあたり限界費用** | **¥0.42** | |

**比較**：
- 海外のH100推論コストは約 **$0.60/100万トークン**（≈ ¥4.3）
- 中国の推論コストは海外の **10%**であり、「価格1/10」という市場現象と一致する
- 国産GPUに置き換えた場合、海外の **7%** まで低下する

### 2.2 販売価格 vs コスト：粗利率の分布

| モデル | 総合ASP (¥/100万) | 限界費用 (¥/100万) | **粗利率** |
|---|---|---|---|
| 国産オープンソース 7B (チームR レベル) | 1.5 | 0.42 | **72%** |
| 国産オープンソース 70B (モデル乙 レベル) | 5.0 | 1.5 | **70%** |
| モデル丙（コーディングトップ） | 20 | 1.5 | **93%** ⭐ |
| モデル甲（動画） | 500/秒 (秒単位課金) | 50/秒 | **90%** |
| 海外トップモデルA | 60 | 30 | **50%** |
| GPT-4o（海外） | 40 | 25 | **38%** |

**重要な発見**：
- **国産汎用LLM**：粗利率 ~70%、**すでにコストを回収できている**
- **国産コーディングトップ**：粗利率 93%、**好循環の極致**
- **動画モデル**：秒単位の課金体系で規模感が全く異なり、それでも粗利率は90%
- **海外主流モデル**：計算リソースコストが10倍高いため、粗利率はより薄い（38-50%）

### 2.3 クローズドループ／好循環の判定基準

**三つの必要条件**（すべて同時に満たす必要あり）：

✅ **条件1**：モデルが特定の垂直分野でトップ（上位3位以内）にあること
✅ **条件2**：顧客のビジネスが深くロックインされていること（移行コスト ＞ 切り替えメリット）
✅ **条件3**：トークン収入 ＞ 計算リソース＋研究開発＋販売コスト（粗利率 ＞ 30% かつ ARR成長率 ＞ 50%）

|---|---|---|---|---|

| 国産オープンソースリーダーチームR | ✅ オープンソース第1位 | ❌ 完全オープンソース・無料 | ⚠️ 海外でクローズド／国内で半クローズド | ⭐⭐⭐⭐ |

| AIスタートアップZ | ⚠️ マルチモーダルに特色あり | ⚠️ オープンソース＋商用 | ⚠️ 動画は単体で利益貢献 | ⭐⭐⭐ |
| AIスタートアップY | ⚠️ 長文テキストでトップ | ⚠️ 個人ユーザーが多い | ❌ 2024年は資金燃焼モデル | ⭐⭐ |
| 大手総合クラウドE / モデル戊 | ❌ 汎用レースで特色なし | ❌ 内部利用が中心 | ❌ 対外売上は限定的 | ⭐⭐ |
| AIスタートアップW | ⚠️ 一部のシーンのみ | ⚠️ 汎用レース | ❌ ごく僅か | ⭐ |

**結論**：
- **クローズドループ達成済み**：特定分野のトップモデル（動画／コーディング）
- **半クローズドループ**：汎用LLMのトップクラス（グループからの資金注入、またはオープンソースのフライホイールに依存）
- **未達成**：汎用LLMのロングテール＋トラフィック型AI企業

**好循環は成立しているのか？**

- **成立している**：モデル向上 → 顧客増加 → データ還流 → さらなるモデル向上（動画／コーディングのトップ企業）
- **部分的に成立**：オープンソースのフライホイール（チームR／モデル乙）— 世界中からの呼び出しがデータとして還流し、国内ユーザーも教育される
- **未成立**：資金を投じてトラフィックを買うAI企業 — 顧客に粘着性はなく、モデルに差別化要因もなく、市場からの撤退は時間の問題

##  3、MaaSの販売パラダイムは学ぶ価値があるか

### 3.1 中国MaaS販売パラダイムの核心

大手総合クラウドBが2025年から2026年にかけて推進したMaaS販売パラダイムの変革：

```
従来のクラウド販売（リソースを売る）
    ↓ 転換
MaaS販売（トークンを売る）
    ↓ 進化
エージェント販売（能力／シーン／成果を売る）
```

**具体的なアクション**：
1.  複数の新たなMaaS専任営業チームを設立し、既存の直販チームと連携して案件を獲得する
2.  **コアKPI**：利用シーンの発掘 ＋ 企業へのエージェント導入促進 ＋ トークン消費量の増加
3.  **関連製品**：業界別エージェントテンプレート（ECカスタマーサポート／金融リスク管理／行政窓口／工業系ドキュメント処理）
4.  **プライシングモデル**：年間契約 ＋ トークンパッケージ ＋ 成果報酬（GMV連動／削減工数連動）

大手動画クラウドAのバリエーション（よりアグレッシブ）：
- モデル甲（動画）では、**ホワイトリスト＋年間契約**モデルを直接採用
- 春節のピーク時には10時間待ちの行列が発生 → 「最も高く買ってくれ、最もビジネスが深い」顧客を強制的に選別
- 「フルスペック版」の安定供給 → GPUリソースを年契約顧客に優先的に割り当て

### 3.2 米国式AI販売との比較

| 次元 | 中国式MaaS販売 | 米国式AI販売 (OpenAI / Anthropic) |
|---|---|---|
| **コアKPI** | シーン数＋トークン消費＋ビジネス成果 | MAU＋ARR＋リテンション |
| **顧客分類** | 業界Key Account（金融／政府・公共／製造業） | 開発者＋企業PM＋IT購買 |
| **リーチ方法** | セールスドリブン＋役員レベルBD | プロダクトドリブン＋セルフサービスドキュメント＋セールスアシスト |
| **プライシングモデル** | 年間契約＋トークンパッケージ＋成果報酬 | 従量課金＋エンタープライズ契約＋ティア別価格 |
| **競争優位の源泉** | グループ内シーンでの囲い込み | モデル能力＋ツールチェーンエコシステム |

### 3.3 「形」と「神髄」の分解

**学べる「形」**（営業組織／プロセス／ツール）：
- ✅ MaaS専任の営業チームを設立し、既存の直販と連携させること
- ✅ KPI設計：シナリオ数＋トークン消費量＋ビジネス成果
- ✅ 業界別エージェントテンプレートライブラリ
- ✅ 年間契約＋トークンパッケージ＋成果報酬というプライシング
- ✅ ホワイトペーパーやROI試算ツールの整備

**学べない「神髄」**（根底にある優位性）：
- ❌ **グループ内シーンでの囲い込み** — AはTikTokショートドラマ（年間220〜300億）、Bは淘宝カスタマーサポート（百万規模の事業者）、CはWeChat公式アカウントを持つ。海外大手にはこのような「自社の養魚池」が存在しない
- ❌ **グループ内でのGPU／電力スケジューリング** — A/B/Cは「自社GPUの遊休時間帯」の計算能力をMaaS事業に割引提供できる
- ❌ **国産GPUへの代替** — 国産Aカードの「使えればOK」という水準により、H100/H200が禁輸された後も、中国大手には推論能力が残されている
- ❌ **データコンプライアンス上の優位性** — 国内の大規模顧客（金融／政府公共／中央企業）は、当然の選択として国内クラウドを選ぶ

### 3.4 誰にとって学ぶ価値があるか

| 企業タイプ | 学ぶ価値があること | 学べないこと |
|---|---|---|
| **産業シーンを持つクラウド事業者** | 全面的に学べる（形＋KPI調整） | 自社の養魚池の構築には時間がかかる |
| **純粋なAIスタートアップ** | 形は学べる（営業組織／プライシングモデル） | 養魚池／計算資源／データコンプライアンスは学べない |
| **従来型の業界ISV/SI** | MaaS販売のKPI体系は学べる | 自社開発モデル／計算資源が不足している |
| **海外大手企業** | 「年間契約＋トークンパッケージ」というプライシングは学べる | 中国大手の「グループによる囲い込み」は学べない |

##  4、価格1/10でも機能し続ける5つのメカニズム

これは本稿で最も硬派なセクションだ。**中国のAPI価格は長期的に海外の10分の1に抑えられ、市場が崩壊していないのは、補助金によるものではなく、5つの基盤的メカニズムが同時にボリュームを牽引し、さらに3つの中国特有の要素が重なっているからだ。**

### メカニズム1：ロングコンテキスト税

**現象**：2025年～2026年にかけて、主要モデルのコンテキストウィンドウが8Kから128K / 1M / 10Mへと飛躍的に拡大した。

- 1回のRAGリクエストに50万～100万文字のコンテキストを詰め込む
- 1回のAPI呼び出しあたりのトークン数が約1Kから約500Kに増加（**500倍**）
- 顧客の体感価格は「変わらない」（API単価が5～10分の1に下がった）が、**実際の消費金額**は50倍に増加

**データ**：
- 2024年：1回あたり平均約2Kトークン
- 2026年：1回あたり平均約150Kトークン
- トークン単価は90％下落したが、**1回あたりの金額は75倍** → 顧客の総支払額は逆に7.5倍増

**勝者**：モデル提供事業者（1回あたりの課金額が増加）＋ 上位アプリケーション（UXが向上）→ ウィンウィン

### メカニズム2：出力インフレ

**現象**：モデルは「安全で冗長な出力」を好む傾向がある。本来50文字で済む内容を、AIは500文字で書く。

- 初期GPT-4：平均出力200トークン/リクエスト
- 2026年Q1の主要モデル：平均出力800トークン/リクエスト
- **出力トークンは4倍に増加**（入力が変わらなくても）

**経済学的意味**：
- 出力トークンの価格は通常、入力の3～5倍（コーディングや動画生成で特に顕著）
- 顧客は「出力トークン」に応じてより多く支払う → 事業者の収入増
- しかし、GPUの実際の計算量は4倍には増えない（キャッシュ / 投機的デコーディング / 量子化）→ 限界コストの伸びは収入の伸びを大きく下回る

**事業者にとって**：粗利率が拡大する

### メカニズム3：スケジューリング・ディスカウント

**現象**：1トークンあたりの限界コストは、以下の4つの技術的手段によって継続的に低下している。

| 技術 | 1トークンあたりのコスト削減 |
|---|---|
| キャッシュ（Cache） | キャッシュヒット → 90％低下 |
| 量子化（Quantization） | FP16 → INT8 / INT4 → 50～75％低下 |
| MoEアーキテクチャ | 活性化パラメータを3～5％に圧縮 → 急降下 |
| 投機的デコーディング | 推論時間が30～50％短縮 |

**データ**（オープンソースリーディングチームRの公開技術レポートより）：
- 1トークンあたり推論コスト 2024年Q4：$0.0014
- 1トークンあたり推論コスト 2026年Q1：$0.0002（**86％低下**）
- API販売価格 2024年Q4：$0.27/100万トークン
- API販売価格 2026年Q1：$0.014/100万トークン（**95％低下**）
- **販売価格の下落幅がコストの下落幅より大きい → 粗利率はむしろ拡大**

### メカニズム4：予算の内製化

**現象**：AI利用が「研究開発プロジェクトの支出」から「経常的な運営支出」に変わる。

- 2024年：CFOの視点＝「これは研究開発部の新しいおもちゃ」
- 2026年：CFOの視点＝「これは水道・光熱費＋SaaSサブスク＋クラウドリソース」

**事業者にとっての意味**：
- 顧客内部の承認プロセスが短縮される（IT部門の決定権が増大）
- 更新率が上昇（業界基準60～80％）
- ARPUが上昇（CFOが自ら予算上限を引き上げる）

**データ**：
- 2024年の企業AI予算の中央値：年間5万ドル
- 2026年の企業AI予算の中央値：年間50万ドル（**10倍増**）
- 主なドライバー：「試験導入」→「本格展開」→「固定費化」の三段跳び

### メカニズム5：コンプライアンス・プレミアム

**現象**：監査可能 / ソース記録付き / プライベートデプロイのトークンは、なお30～100％のプレミアムがつく。

- 金融業界：監査用トークンの価格は標準の**1.3倍～2.0倍**
- 政府・公共機関向け：プライベートデプロイのトークン価格は標準の**1.5倍～3.0倍**
- 医療 / 法務：トレーサビリティのあるトークン価格は標準の**1.2倍～1.5倍**

**事業者にとっての意味**：
- 標準APIの価格競争は激しい（粗利率30～50％）
- コンプライアンス / プライベートデプロイ / 業界特化トークンは引き続き高粗利（粗利率60～80％）
- **MaaS販売の核心的利益源**は、標準APIではなく、このコンプライアンス・プレミアムにある

### 中国特有の3つの重なり

| 中国特有の要素 | 節約 / 上乗せ効果 | メカニズム |
|---|---|---|
| **国産GPUによる代替** | 推論コスト30～50％低下 | 国産Aカードの価格はH100の50～70％、TDPはほぼ同等 |
| **電力補助金** | 推論コスト15～25％低下 | 内モンゴル / 貴州 / 寧夏のIDC電気代は0.3～0.4元/kWh、沿海部より30～50％安い |
| **オープンソース・エコシステムのフライホイール** | グローバル呼び出しが国内固定費を薄める | オープンソースリーディングチームR / モデル乙がOpenRouterのシェア50％超を獲得 → 海外呼び出しの還流 → 国内計算リソースの遊休時間を埋める → 国内ASPは下がるが総粗利は増加 |

**純効果**：
- 中国の推論コスト ＝ 海外の10～15％（構造的）
- 中国のAPI販売価格 ＝ 海外の10～15％（追随）
- **1トークンあたりの粗利率は海外と同等か、むしろ高い**（構造的コスト低＋コンプライアンス・プレミアム強）

##  5、中国MaaSの3つの独自戦略

### 戦略1：シナリオという生簀で育てる → トークン刈り取り

1. TikTok系ショートドラマ事業で軌道に乗る → モデル甲の能力を実証

5. **MaaS事業の四半期収入は約30億元**（四半期AI収入の30％と推定）

### 戦略2：オープンソース・フライホイール → グローバル呼び出しで国内コストを薄める

4. 海外からの呼び出し量 → 国内GPU稼働率が向上
5. **国内価格は低くとも、グローバル呼び出しがトレーニング＋推論＋研究開発の固定費を薄める**

##  6、反対意見 / リスク要因

### 6.1 価格1/10でも機能し続ける3つの不安材料

**不安1：補助金競争はいつまで続くか？**
- 現在の中国API価格は、コスト＋薄利（真の市場清算価格ではない）
- いずれ業界が淘汰され（寡占2～3社に）、価格が5～10倍に跳ね上がる可能性がある
- **類例**：中国クラウド市場、2018～2020年の価格競争 → 2021～2022年に30～50％値上げ

**不安2：オープンソース・フライホイールは持続可能か？**
- オープンソースリーディングチームR / モデル乙は、現在「愛情＋国家戦略的支援」で動いている
- 政策支援が弱まる、あるいはチームの商業化圧力が高まれば → オープンソースライセンスが厳しくなる可能性がある
- **類例**：Red Hat / MongoDB / Elasticは、いずれも「オープンソース → 商業化による制限強化」の道を辿った

**不安3：コンプライアンス・プレミアムの持続可能性**
- 現在、コンプライアンス / プライベートデプロイの30～100％プレミアムは、「国産代替＋データセキュリティ」政策に基づく
- 海外モデルの中国再参入が認められれば → コンプライアンス・プレミアムは圧縮される

### 6.2 ビジネスループの3つの脆弱点

**脆弱点1：単一モデルのウィンドウ期間**
- モデル甲は現在、ショートドラマ業界でシェア95％ → 業界は飽和状態
- 海外動画モデルが3世代進化して追随 → バイトダンスの堀はいつまで持つか？
- **見立て**：動画モデルのウィンドウ期間は18～24か月、次世代モデルへの継続投資が不可欠

**脆弱点2：コーディングモデルの同質化**
- モデル丙は現在コーディングでトップだが、海外モデル＋Cursor / Devinがシェアを奪っている
- 海外モデルが再参入を認められれば、コーディングの首位は交代する可能性がある
- **見立て**：コーディングのウィンドウ期間は12～18か月、開発者エコシステムとの結びつきが必須

**脆弱点3：汎用LLMの価格競争泥沼**
- 国産オープンソース7B / 70Bの価格は既に0.5～2元/100万トークンまで低下
- これ以上の値下げ余地は限定的（限界コストに近づきつつある）
- **見立て**：汎用LLMレースは2年以内に3～5社の寡占状態となり、価格競争は終結する

##  7、結論：ビジネスループが成立する条件

### 7.1 成立する4つの条件（すべて同時に満たす必要あり）

✅ **条件1**：モデルが何らかの垂直シナリオでトップ（上位3社）に入っていること
✅ **条件2**：顧客の業務と深く結びついていること（スイッチングコスト > 切り替えによる利益）
✅ **条件3**：トークン収入 > 計算リソース＋研究開発＋販売コスト（粗利率30％超）
✅ **条件4**：「自社の生簀」または「オープンソース・フライホイール」または「コンプライアンス・プレミアム」のいずれかを持っていること

### 7.2 ビジネスモデルの比較：海外 vs. 中国

| 次元 | 海外（OpenAI / Anthropic） | 中国（某大手動画クラウドA / コーディングトップX / 総合クラウドB） |
|---|---|---|
| **主要シナリオ** | 開発者＋個人向けPro | グループ内シナリオ＋業界KA |
| **ループ経路** | モデル性能＋ツールチェーン＋エコシステム | グループ内生簀育成＋オープンソースフライホイール＋コンプライアンスプレミアム |
| **堀** | モデル品質＋ツールチェーン | 自社シナリオ＋データコンプライアンス＋国産化 |
| **粗利率** | 38～50％ | 70～93％（トップ） / 30～50％（汎用） |
| **価格競争へのスタンス** | 慎重、世界的な価格体系を崩さない | 積極的、国内で先に勝ち切る |
| **持続可能性** | 高い（CFOの受容性が強い） | 中程度（政策＋オープンソースエコシステムに依存） |

### 7.3 読者への判断材料

**中国のMaaSプレイヤーへ**：
- 汎用LLMレースは18か月以内に淘汰される
- 垂直シナリオ（動画 / コーディング / ショートドラマ / 業界エージェント）こそ、2026～2028年に真に稼げる主戦場
- 「自社の生簀」が究極の堀。グループ内シナリオで育て → トークンを刈り取る経路は、全面的に学ぶ価値がある

**顧客 / 投資家へ**：
- 「値上げしても量が落ちない」はトップAIモデルで成立する（コーディングトップXが既に実証済み）
- 1/10の価格はあと2～3年持続し、その後5～10倍に跳ね上がる（業界淘汰）
- オープンソースフライホイール ＝ グローバル呼び出しが国内コストを薄める。**オープンソースリーディングチームR / モデル乙は構造的勝者**

**海外読者へ**：
- 中国MaaSの販売パラダイムは「形は学べても神髄は学べない」
- 真の堀は、グループ内シナリオという生簀で育てること——これは中国インターネット大手の独自資産
- 海外は「年間契約＋トークンパック＋成果連動」という価格モデルは学べるが、「自社の生簀」は手に入らない
