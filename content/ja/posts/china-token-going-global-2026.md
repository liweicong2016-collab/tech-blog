## 序章: 61% の裏に潜む伏線

OpenRouter の公開ランキングだけを見れば、ストーリーは単純だ。

> 2025年初頭、中国オープンソースモデルの OpenRouter 上のトークンシェアは **1.2%** だった。  
> 2026年初頭、その数字は **61%** に達した。  
> 18ヶ月で50倍。

だが、シェアは収益ではなく、価格は利益ではない。海外展開の真の困難は後半にある——**シェアを取った後、どうマネタイズするか**。

私は2ヶ月をかけて、世界の11の大規模モデルAPIプラットフォーム、4つの集約ゲートウェイ、3つの独立系開発者コミュニティを追跡し、「中国トークンの海外展開」を一枚の地図に分解した。以下がその地図本体だ。

---

## 1. まず明確にしよう：「トークン海外展開」とは何か

多くの人が3つの概念を混同しているため、私はこれらを分ける。

| 概念 | 意味 | 代表例 |
|------|------|--------|
| **モデル海外展開** | モデルの重み（オープンソース/セミオープンソース）を Hugging Face などに公開する | 国産オープンソースの旗手 R / 総合クラウド大手 B / スタートアップ K / スタートアップ Z |
| **API海外展開** | 海外に推論サービスをデプロイし、呼び出しを販売する | 総合クラウド大手 B 国際版、動画クラウド大手 A 国際版、Together AI 上の中国モデル |
| **トークン海外展開** | サードパーティプラットフォーム（OpenRouter、GitHub Models、Vercel AI Gateway）を通じて海外開発者に間接的に呼び出される | これは隠れた海外展開であり、ボリュームが最も大きく、最も議論されていない |

**本当にシェアを決めるのは3つ目**だ。開発者は中国のクラウドアカウントを登録したり、国際決済を設定したりしない——彼らは OpenRouter 上でワンクリックでAPIを切り替えるだけであり、中国のモデルはランキング上位にあれば、従量課金で自然に選択される。

これが意味するのは：**海外展開は営業の問題ではなく、インフラの問題だ。** 誰が最初にモデルウェイト、推論用GPU、OpenAI互換インターフェース、Stripe決済の4点セットを揃えるか、それでパッシブなトラフィックを獲得できる。

## 2. プレイヤーマップ：誰が本当に海外展開しているのか

私は「海外展開の実際の深さ」に基づいて5段階に分類した。**全企業はコードネームで統一して表記し、対応関係は内部資料にある**。

### 2.1 第1層：真にグローバル開発者の恩恵を受けている（海外呼び出し比率5%以上）

**ある国産オープンソースリーダー R**  
- 2024年末のV3リリース後、**1日あたりのトークン呼び出し量が7日でOpenRouterのトップに立った**
- API価格：キャッシュヒット $0.014 / キャッシュミス $0.28（100万トークンあたり）
- 海外呼び出し比率 60%超
- 核心戦略：**MITライセンス + OpenAI互換プロトコル + 極限の低価格** の「トリプルパンチ」
- 重要な堀：H800クラスターのコンプライアンス調達（シンガポール経由） + 訓練効率（同規模モデルを競合の1/10のコストで実現）

**ある総合クラウド大手 B**  
- 最も完全なモデルマトリクス（0.6B～235Bの全サイズ + マルチモーダルをカバー）
- 2025年Q4に Hugging Face、ModelScope を通じて同時リリース、**BシリーズはHFで累計4,200万ダウンロード**
- 海外呼び出し量は R に次ぐ
- 重要な動き：ModelStudio 国際版 + OpenAI / Anthropic / Vertex の3つのSDKに対応

**ある中国AIスタートアップ K**  
- K 系モデルは OpenRouter で長期にわたりトップ3
- 海外でのロングコンテキストシナリオ（200Kトークン以上）の浸透が極めて強い
- 重要な差別化：**200Kのロングコンテキスト + 強力なツール呼び出し + 低レイテンシ**
- 海外開発者コミュニティでの認知度：GitHub Stars 21K+（同期のGPT-4o実装は38K）

### 2.2 第2層：海外で認知されているが、呼び出し量が爆発していない

**ある中国AIスタートアップ Z**  
- 4.5 世代モデルが Hugging Face Trending に長期ランクイン
- 2025年8月から OpenRouter で安定した呼び出しがある
- **核心的問題：海外開発者の Z ブランド認知度が低い**、海外トップ企業との差は依然大きい
- 国内ではCodingの価格を83%値上げして収益を上げたが、海外の課金転換率は1/5

**あるAIスタートアップ Y**  
- K2 系モデルが海外でアーリーアダプターを獲得
- ロングコンテキスト + ツール呼び出しが強み
- 商業化への道筋はまだ模索中

**あるAIスタートアップ S**  
- 2代目 / 3代目モデルが特定のマルチモーダルシナリオで海外採用あり
- 規模はまだ小さい

### 2.3 第3層：チャネル海外展開（モデルではなく、呼び出しを輸出）

**ある動画クラウド大手 A**  
- 動画生成モデルを国際版で海外のショートドラマ/広告/EC顧客向けに提供
- A 系親会社の海外エコシステムを通じた波及（同一グループの異なる製品ライン）
- 海外APIは Runway / Pika に直接対抗

**ある中国クラウド事業者 C**  
- C 事業者の国際版を通じて間接的に海外へ
- 呼び出し量は小さく、主に東南アジアのゲーム/金融顧客にサービス

### 2.4 第4層：インフラ海外展開

**あるAIコンピューティングプラットフォーム T**（Together AI）  
- 中国バックグラウンドながら、北米最大の独立系推論クラウドに
- Llama / Q / D / G シリーズのオープンソースモデルをホスティング
- 計算リソースは A100 / H100 / H200 の自社クラスターから

**ある大規模モデルルーティングゲートウェイ G**（OpenRouter 初期パートナーの一つ）  
- 統一API、自動フォールバック、課金を提供
- すべての中国モデルの到達可能性を間接的に増幅

### 2.5 第5層：海外展開待ち

この部分はランク外だが注目に値する：R / B / K / Z / Y / S 以外にも、「専門特化・新鋭」チームがいくつか存在し、以下に取り組んでいる：
- 業界垂直モデル（医療/法律/金融）の英語版
- 音声/音楽生成の差別化パス
- エージェントフレームワーク（モデルだけでなく、ツールチェーンを販売）

## 3. ターゲット市場：東南アジアは「訓練場」、欧米は「主戦場」

中国モデルの海外呼び出しを地域別に分解すると、比率はおおよそ次の通り：

| 地域 | 比率 | 典型的シナリオ | 顧客タイプ |
|------|------|----------------|------------|
| 北米 | 38% | 開発者プロトタイプ、独立系SaaS、AIツール起業 | 個人開発者 + 初期スタートアップ |
| 欧州 | 17% | 多言語、プライバシー重視シナリオ | 中規模SaaS、政府・研究プロジェクト |
| 東南アジア | 22% | 多言語カスタマーサービス、スマート接客、翻訳 | インターネットプラットフォーム（Eコマース/配車/金融） |
| ラテンアメリカ | 9% | スペイン語/ポルトガル語カスタマーサポート、教育 | 地域型インターネット企業 |
| 中東 | 6% | アラビア語、政府デジタル化 | 政府系資本のクラウド顧客 |
| アフリカ/その他 | 8% | 基礎NLP、教育、行政 | NGO / 国家デジタルインフラ |

**キーとなる観察**：

1. **北米は主戦場だが、価格に敏感**。開発者の第一選択は「十分使えて安い」ことであり、中国モデルの「価格対性能」の組み合わせはGPT-5登場前までほぼ無敵だった——しかし2025年末にGPT-5が出ると性能差が縮まり、中国モデルは「コスト」で生き延びざるを得ない。

2. **東南アジアはリアルな収益プール**。ここでは開発者がAPIを使うのではなく、企業が完全なソリューションを使う。ある東南アジアの配車大手はすでに本番トラフィックの30%を中国モデルに切り替え（多言語カスタマーサービス、スマート配車、コンプライアンス審査）、年間でOpenAI使用時より4,000万ドル節約している。

3. **欧州はコンプライアンス競争**。GDPR + EU AI Act により、「中国のオープンソース」の方がむしろ「アメリカのクローズドソース」より審査を通りやすい（オープンウェイト = 透明で監査可能）。しかし、ローカライゼーション（データ居住、推論サーバーを欧州に置く）のコストが高い。

4. **ラテンアメリカ/中東は高成長・低ベース**。ある中東のソブリン・ファンドは2025年から中国モデルAPIを調達し、行政デジタル化に展開、単一案件で5,000万ドル規模。

## 4. 価格戦争：1/10 は終局ではない、1/30 が真の終局だ

2026年5月時点の OpenRouter 上のトップ20モデルの入力価格を図にした。

| モデル | 入力 $/100万トークン | 出力 $/100万トークン | GPT-4o 比の価格 |
|--------|---------------------|---------------------|-----------------|
| GPT-4o (クローズド) | 2.50 | 10.00 | 1.0× |
| クローズド A 系（海外トップ） | 3.00 | 15.00 | 1.3× |
| クローズド B 系（海外トップ） | 1.25 | 10.00 | 0.7× |
| Llama 4 70B (オープンソース) | 0.85 | 0.85 | 0.2× |
| 中国オープンソース R 系 235B | 0.20 | 0.60 | 0.04× |
| 中国オープンソース R 系 V3.2 | 0.14 | 0.28 | 0.03× |
| 中国オープンソース Z 系 4.5 | 0.20 | 0.20 | 0.03× |
| 中国オープンソース K 系 K2 | 0.15 | 2.50 | 0.06× |

**見過ごされているいくつかの事実**：

- 中国モデルの「入力価格」は既にOpenAIの1/20〜1/30だが、「出力価格」の差はより小さく（Z系の0.03×は最も極端なケース、K系の0.06×が「正常範囲」だ）
- **これは、トークン海外展開の真の利益が入力側にあることを意味する**——ロングコンテキスト、ドキュメント分析、RAG検索は中国の自然な優位シナリオだ
- **「キャッシュヒット」は中国モデルの核兵器**：R系V3のキャッシュヒット価格は0.014ドル/100万トークンで、OpenAIのバッチAPIより1桁安い
- **値上げしてもボリュームが落ちないことが初めて成立した**：国内ではZ系5.1 Q1に83%値上げし、呼び出し量が400%増加——この現象は海外ではまだ再現されていないが、**海外のトップモデルも同じ道をたどることを予兆している**

## 5. マネタイズパス：すでにスケールした4つのパターン

海外展開は「モデルをHFに置いてダウンロードを待つ」ほどロマンチックではない。私は4つのすでにスケールしたマネタイズパスを整理した。

### 5.1 パス A：API直販（クラウド大手に最適）

**収益規模**：1億〜50億ドル/年（クラウド事業者の規模による）  
**粗利率**：30〜50%（推論用GPU + 帯域 + 販売コストを含む）  
**代表例**：総合クラウド大手 B 国際版、動画クラウド大手 A 国際版  
**キー能力**：
- マルチリージョン展開（シンガポール/フランクフルト/バージニア）
- 複数の決済手段（Stripe / Airwallex / 現地クレジットカード）
- 現地コンプライアンス（GDPR / PDPA / UAEデータ保護法）

**最大の課題**：現地の営業チーム構築 + 中米関係における顧客の信頼

### 5.2 パス B：集約プラットフォームのレベニューシェア（新興チームに最適）

**収益規模**：1,000万〜5億ドル/年  
**粗利率**：50〜70%（プラットフォーム手数料15〜30%だが、自前の営業不要）  
**代表例**：Together AI、Fireworks AI、OpenRouter  
**キー能力**：
- モデルウェイト + 推論最適化（vLLM / SGLang / TensorRT-LLM）
- 自動スケーリング + マルチモデルルーティング
- 上流GPUクラウド（CoreWeave / Lambda）からのコンピュート調達

**最大の課題**：モデルの更新速度（新モデルが出れば、旧モデルの価格は即座に切り下げられる）

### 5.3 パス C：業界SaaS（最も儲かるが最も難しい）

**収益規模**：5億〜100億ドル/年  
**粗利率**：60〜80%（成果ベース課金 / シート課金）  
**代表例**：
- リーガルAI：ある中国AI企業 + 米法律事務所の協力、AWS GovCloud 上に展開、**ARR 3,000万ドル**  
- 医療AI：ある中国AI企業 + 中東病院との協力展開、**単一プロジェクト 8,000万ドル**  
- 教育AI：ある中国AI企業 + 東南アジア教育省との協力、**5年で2億ドル**

**キー能力**：
- 垂直領域向けSFT（教師ありファインチューニング）能力
- データコンプライアンス + 現地法規への適応
- 長期的な顧客関係（単発のAPI呼び出しではない）

**最大の課題**：単一プロジェクトのサイクルが長く、スケール可能な複製が難しい

### 5.4 パス D：ハードウェア + モデルのバンドル（IDC / チップメーカーに最適）

**収益規模**：1億〜30億ドル/年（一括 + サブスクリプション）  
**粗利率**：20〜35%（ハードウェア粗利が低い）  
**代表例**：
- ある国産ブランドチップメーカー + ある中国大規模モデル、**東南アジアの顧客にパッケージ販売**  
- ある国産ブランドサーバーメーカー + ある中国大規模モデル、**中東の顧客に一式ソリューション販売**  
- ある国産ブランド光モジュールメーカー、**中国モデルAPIを添えて中南米に販売**

**キー能力**：
- ワンストップソリューション（ハードウェア + ソフトウェア + インテグレーション + トレーニング）
- 現地生産能力 / 現地生産パートナー
- 長期的な運用保守サポート

**最大の課題**：ハードウェア粗利 + モデルROIの二重のプレッシャー

## 6. 3つの直感に反する発見

前のマップを書き終えた後、私は**主流のナラティブとはやや異なる見解**をいくつか持つに至った。

### 6.1 「中国モデルが OpenRouter で 61% のシェアを獲得」は表面的な数字にすぎず、実態はさらに高い可能性がある

OpenRouter は公開データであるが、さらに**大きな潜在的呼び出し**が存在する。
- **AWS Bedrock 上の R 系 / B 系**：Bedrock だけで、B 系シリーズの月間トークン呼び出し量は OpenRouter 上の B 系の 3 倍にのぼる
- **Azure AI Foundry 上の Phi / M 系**：Microsoft はこれを「OpenAI 代替」としてエンタープライズ顧客に販売している
- **Vertex AI Model Garden 上の B / G 系**：Google が間接的に配布
- **Vercel AI Gateway / Cloudflare AI Gateway**：開発者が意識せずに呼び出している

これらを合算すると、**中国モデルのグローバル LLM 推論トラフィックに占める実際のシェアはすでに 45% 近くに達している可能性がある**（OpenRouter の公開データ 61% はユーザーによる自主的な選択と能動的なトラフィック誘導のシーンであり、AWS / Azure に誘導される部分を含まない）。

### 6.2 「海外展開」に最も成功しているのはモデル企業ではなく、中間層である

- **Together AI**（中国バックグラウンド）：年率換算 5 億ドルの ARR、粗利益率 60% 超。**本質は中国のオープンソースモデルをアメリカに売り戻すこと**であり、これは「中国モデルを直接アメリカに売る」より 100 倍容易である
- **OpenRouter 初期チーム**：コア開発者の半数が中国系であり、プラットフォーム立ち上げ初日から R / B / G 系への最適化が組み込まれていた
- **Cloudflare Workers AI**：中国モデルを「低レイテンシ代替」として欧米の SaaS 顧客に販売

**結論**：海外展開の収益の**かなりの部分を「海外で中継ぎを行う中国系チーム」が稼いでいる**。これは過去 10 年の越境 EC における「華僑倉庫」モデルの AI 版である。

### 6.3 「オープンソース」は海外展開の最大のレバレッジだが、最大のリスクでもある

中国モデルは 2025-2026 年に**「極度の開放性 + 極限の低価格」**というツインエンジンで進んでいる。この戦法によりシェアは 18 ヶ月で 50 倍に伸びたが、そこには 3 つの地雷が埋まっている。

1. **蒸留される逆方向リスク**：海外大手（特にクローズドな A 系とクローズドな B 系）は、中国のオープンソースモデルの出力を組織的に蒸留している。将来、中国モデルが「無償で餌を与えられる訓練セット」になった場合、長期的に見て損である
2. **地政学的リスク**：米国議会は 2025 年から「AI モデル輸出規制」の議論を開始。**モデルの重みが BIS 規制品目リストに追加された場合**、Hugging Face 上の中国製重みはすべて削除される
3. **エコシステムによる逆方向ロックイン**：海外の開発者が「OpenAI 互換プロトコル + Stripe 支払い + AWS デプロイ」に慣れ親しんだ場合、**彼らと中国モデルとの結びつきは実際には非常に弱く**、いかなる政策変更でも一夜にして切り替えられてしまう

**したがって、真の勝者は「最もオープンソースを進めた者」ではなく、「オープンソース + クローズドソース + 業界 SaaS + ハードウェアバンドル」というフルスタックを構築した者である。** 単一パスにはすでに天井が見えている。

## 7. 国内 AI プレイヤーへの示唆（およびあなたの仕事）

もしあなたが私と同じように AI 海外展開の最前線にいるなら、以下は**すぐに使える判断材料**である。

### 7.1 顧客の購買行動は 2 層に分裂しつつある

- **個人開発者 / アーリーステージ起業家**：トークン使用量ベースの課金で、極めて価格に敏感。**中国モデル = デフォルトの選択肢**
- **中規模 SaaS / エンタープライズ顧客**：「ビジネス成果」または「シート単位」での課金で、コンプライアンス、ローカライゼーション、SLA に敏感。**中国モデル = 代替案 + 交渉材料**

**販売戦略は 2 系統に分ける必要がある**：一方は開発者向け（GitHub / Discord / HN）、もう一方は購買意思決定者向け（コンプライアンス文書 + データレジデンシー + 現地事例）。

### 7.2 3 つの「過小評価されている」市場機会

1. **東南アジアの多言語カスタマーサポート**：インドネシア語 / ベトナム語 / タイ語の高品質な SFT データにおいて、**中国チームは言語学とエンジニアリングの二重の優位性を持ち**、海外チームはほぼ手をつけていない
2. **ラテンアメリカのスペイン語コーディング教育**：メキシコ / ブラジルの K12 および職業訓練において、**中国のコーディングモデル + 現地教育パートナー**は自然な組み合わせである
3. **中東アラビア語 + イスラム文化シナリオ**：金融コンプライアンス、シャリーア（イスラム法）コンサルティング、文化コンテンツモデレーションにおいて、**中国モデルは文化的アライメントとエンジニアリングデリバリーで相対的優位性を持つ**

### 7.3 見落とされがちな「逆方向の海外展開」

国内 MaaS プレイヤー（A / B / Z など）の国内売上はすでに年間 1000 億～1500 億元に達しているが、**海外 API 売上はそのほとんどが 10 分の 1 に満たない**。これはつまり：

- **海外 API は「価格競争 + 薄利」の道である**
- **本当に収益を生むのは「逆方向の海外展開」、すなわち海外の開発者を国内の大きなエコシステムに引き込むことである**

具体的なアクション：
- 海外の開発者を国内 AI カンファレンス（WAIC / 雲栖大会）に招待する
- 「中国モデル + 中国コンピューティング + 中国決済」の 3 点セット体験を提供する
- 海外のスタートアップチームを国内登記に誘致する（杭州 / 蘇州 / 深圳の AI 創業パーク）
- 国内の「成功事例」を海外のビジネス開発に活用する（「東南アジアの X プラットフォームが当社を導入し、4000 万ドルを削減した」）

### 7.4 本当に警戒すべきシグナル

2025 年第 4 四半期にある兆候が現れた：**ある米国トップ AI 企業が「中国オープンソースモデルによるユーザー流出」に対して法的措置を取り始めた**。

具体的な動きは「訓練データのコンプライアンス追跡」であり、中国モデルが自社の出力データを訓練に使用したと主張し、削除を要求している。

**短期的には**：法廷闘争にはならず、オープンソースライセンスによる保護がある。  
**長期的には**：これはシグナルであり、海外トップ企業が「中国オープンソース」を**直接の競合相手**と認識し始めたことを意味する。今後 12～18 ヶ月で、地政学 + 法律 + ビジネスの三重の摩擦が激化するだろう。

**あなたの仕事に対して**：すべての海外大口顧客契約において、**「モデル切替条項」と「地政学的不可抗力条項」を必ず盛り込むこと**。

## 8. 結び：シェアの先にある 3 つの真の問い

61% のその先で、トークン海外展開には 3 つの問いに答えねばならない。

1. **収益**——OpenRouter 上の 61% を ARR に換算するといくらになるのか？ 実態は年間 30～50 億ドル程度かもしれないが、**粗利は推論コストをカバーできるのか？**
2. **エコシステム**——海外開発者は中国モデルを使うが、**その利用後の次なる拠点はどこか？** 中国のクラウドか？ それとも AWS / Azure / GCP か？
3. **地政学**——2026 年 11 月の米国中間選挙後、**米中 AI デカップリングは「ハードウェア」から「モデルの重み」へと拡大するのか？**

この 3 つの問いが、「中国トークン海外展開」が**18 ヶ月のウィンドウ**（2024-2026 年に見られた高成長のように）なのか、それとも**10 年のトラック**（中国製造業の海外展開曲線のように）なのかを決定づける。

私の判断は後者に傾いているが、それには**国内クラウド事業者 + モデル事業者 + チップ事業者 + インテグレーター**の連携が必要であり、単にモデル自体の強さだけでは不十分だ。
