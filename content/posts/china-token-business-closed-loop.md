## 二、闭环 / 正循环的硬算账

### 2.1 单 Token 成本结构（推理侧，2026-06 反推）

按中国主流 GPU 服务器 + 电力成本反推：

| 成本项 | 数值 | 备注 |
|---|---|---|
| GPU 服务器采购 | ¥350,000/台 | 8 卡 H100 / 8 卡 H200 / 8 卡国产A卡 |
| 服务器折旧 | 3 年线性 | 月 ¥9,720 |
| 电费 | ¥0.65/kWh（含税/含补贴） | 上海/北京/深圳 IDC |
| 单台满载功耗 | 5.6 kW | 8 卡 + CPU + 散热 |
| 单台月电费 | ¥2,624 | |
| 运维 + 机房机位 | ¥3,000/台/月 | |
| 带宽 + 存储 | ¥1,500/台/月 | |
| 人员摊销 | ¥4,000/台/月 | |
| **单台月总成本** | **¥20,844** | |
| 单台月推理 token 产能 | 50B tokens | 假设 30% 利用率 |
| **单 token 边际成本** | **¥0.42 / 百万 tokens** | |

**对比**：
- 海外 H100 推理成本约 **$0.60 / 百万 tokens**（≈ ¥4.3）
- 中国推理成本 = 海外 **10%**，与"价格 1/10"市场现象一致
- 国产 GPU 替代后 = 海外 **7%**

### 2.2 售价 vs 成本：毛利率分布

| 模型 | 综合 ASP (¥/百万) | 边际成本 (¥/百万) | **毛利率** |
|---|---|---|---|
| 国产开源 7B (开源领军团队 R 级别) | 1.5 | 0.42 | **72%** |
| 国产开源 70B (模型乙 级别) | 5.0 | 1.5 | **70%** |
| 模型丙（Coding 头部） | 20 | 1.5 | **93%** ⭐ |
| 模型甲（视频） | 500/秒 (视频按秒计费) | 50/秒 | **90%** |
| 海外头部模型 A（海外） | 60 | 30 | **50%** |
| GPT-4o（海外） | 40 | 25 | **38%** |

**关键发现**：
- **国产通用 LLM**：毛利率 ~70%，**已经打平成本**
- **国产 Coding 头部**：毛利率 93%，**正循环极致**
- **视频模型**：按秒计费，量级完全不同，毛利率仍 90%
- **海外主流**：毛利率更薄（38-50%），因为算力成本高 10 倍

### 2.3 闭环 / 正循环的判定标准

**三个必要条件**（必须同时满足）：

✅ **条件 1**：模型在某个垂直场景做到头部（前 3 名）
✅ **条件 2**：客户业务深度绑定（迁移成本 > 切换收益）
✅ **条件 3**：Token 收入 > 算力 + 研发 + 销售成本（毛利率 > 30% + ARR 增速 > 50%）

| 某头部综合云 B / 模型乙 | ✅ 开源第 2 | ⚠️ 开源可迁移 | ⚠️ 价格战激烈 | ⭐⭐⭐⭐ |
| 国产开源领军团队 R | ✅ 开源第 1 | ❌ 完全开源免费 | ⚠️ 海外闭环 / 国内半闭环 | ⭐⭐⭐⭐ |

| 某头部综合云 E / 模型戊 | ❌ 通用赛道无特色 | ❌ 内部用为主 | ❌ 对外营收有限 | ⭐⭐ |

**结论**：
- **已闭环**：单点头部模型（视频 / Coding）
- **半闭环**：通用 LLM 头部（靠集团输血或开源飞轮）
- **未闭环**：通用 LLM 长尾 + 流量型 AI 公司

**正循环是否成立？**

- **已成立**：模型越好 → 客户越多 → 数据回流 → 模型更好（视频 / Coding 头部）
- **部分成立**：开源飞轮（开源领军团队 R / 模型乙）— 全球调用回流，国内用户也被教育
- **未成立**：烧钱换流量的 AI 公司 — 客户没黏性，模型没差异化，退出市场是时间问题

## 三、MaaS 销售范式是否值得学

### 3.1 中国 MaaS 销售范式的内核

某头部综合云 B 2025-2026 推动的 MaaS 销售范式升级：

```
传统云销售（卖资源） 
    ↓ 转型
MaaS 销售（卖 Token） 
    ↓ 进阶
Agent 销售（卖能力 / 卖场景 / 卖结果）
```

**核心动作**：
1. 成立数个新 MaaS 销售团队，与原直销团队合作打单
2. **核心 KPI**：挖掘使用场景 + 推动企业用 Agent + 增加 Token 消耗
3. **配套产品**：行业 Agent 模板（电商客服 / 金融风控 / 政务问答 / 工业文档）
4. **定价模型**：年框 + Token 包 + 效果分成（按 GMV / 按节省工时）

某头部视频云 A 的变体（更激进）：
- 模型甲（视频）直接**白名单 + 年框**模式
- 春节高峰期排队 10 小时 → 强行筛选"出价最高 + 业务最深"的客户
- "满血版"稳定供应 → 把 GPU 资源优先给年框客户

### 3.2 与美式 AI 销售范式对比

| 维度 | 中国 MaaS 销售 | 美式 AI 销售 (OpenAI / Anthropic) |
|---|---|---|
| **核心 KPI** | 场景 + Token 消耗 + 业务结果 | 月活 + ARR + 留存 |
| **客户分类** | 行业 KA（金融 / 政企 / 制造业） | 开发者 + 企业 PM + IT 采购 |
| **触达路径** | 销售驱动 + 高层 BD | 产品驱动 + 自助文档 + 销售辅助 |
| **定价模型** | 年框 + Token 包 + 效果分成 | 按量计费 + Enterprise 合同 + Tier 定价 |
| **护城河** | 集团内场景养鱼 | 模型能力 + 工具链生态 |

### 3.3 "形 vs 神"的拆解

**可学的形**（销售组织 / 流程 / 工具）：
- ✅ 成立专门 MaaS 销售团队，与原直销合作
- ✅ KPI 设计：场景数 + Token 消耗 + 业务结果
- ✅ 行业 Agent 模板库
- ✅ 年框 + Token 包 + 效果分成定价
- ✅ 配套白皮书 + ROI 测算工具

**学不到的神**（根基性优势）：
- ❌ **集团内场景养鱼** — 某头部视频云 A 有抖音短剧（年 220-300 亿），某头部综合云 B 有淘宝客服（百万商家），某头部综合云 C 有公众号。海外大厂没有这种"自家鱼塘"
- ❌ **集团内 GPU / 电力调度** — 某头部视频云 A / 某头部综合云 B / 某头部综合云 C 可以把"自家 GPU 闲置时段"折价给 MaaS 业务
- ❌ **国产 GPU 替代** — 国产 A 卡的"可用即可"让中国大厂在 H100 / H200 被禁后仍有推理能力
- ❌ **数据合规优势** — 国内大客户（金融 / 政企 / 央企）天然选国内云

### 3.4 对谁值得学

| 公司类型 | 值得学什么 | 学不到什么 |
|---|---|---|
| **有产业场景的云厂商** | 全盘学（形 + 调整 KPI） | 自家鱼塘需要时间建 |
| **纯 AI 创业公司** | 形可学（销售组织 / 定价模型） | 鱼塘 / 算力 / 数据合规学不到 |
| **传统行业 ISV / SI** | 学 MaaS 销售 KPI 体系 | 缺自研模型 / 缺算力 |
| **海外大厂** | 学"年框 + Token 包"定价 | 中国大厂的"集团养鱼"学不到 |

## 四、价格 1/10 还能运转的 5 个机制

这是本文最硬核的一节。**中国 API 价格长期低于海外 10 倍，市场没崩，不是靠补贴，是 5 个底层机制同时拉量 + 3 个中国特色叠加。**

### 机制 1：长上下文税

**现象**：2025-2026 主流模型上下文窗口从 8K 跃迁到 128K / 1M / 10M。

- 一次 RAG 请求塞 50-100 万字上下文
- 单次 API 调用 token 数从 ~1K 涨到 ~500K（**500 倍**）
- 客户感知"价格没变"（API 单价降了 5-10 倍），但**实际消费金额**涨 50 倍

**数据**：
- 2024 年单次 API 调用平均 ~2K tokens
- 2026 年单次 API 调用平均 ~150K tokens
- token 单价跌 90%，但**单次金额涨 75 倍** → 客户总账单反而涨 7.5 倍

**谁赚到了**：模型厂商（单次收费更多）+ 上层应用（用户体验更好）→ 双赢

### 机制 2：输出通胀

**现象**：模型倾向"安全冗长输出"。一段本可以 50 字说清的话，AI 会写 500 字。

- 早期 GPT-4：平均输出 200 tokens / 请求
- 2026 Q1 主流模型：平均输出 800 tokens / 请求
- **输出 token 涨 4 倍**（即便输入没变）

**经济学含义**：
- 输出 token 价格通常是输入的 3-5 倍（Coding / 视频生成尤其）
- 客户按"输出 token"付费更多 → 厂商收入增长
- 但 GPU 实际计算量没有 4 倍增长（缓存 / 投机解码 / 量化）→ 边际成本增速远低于收入增速

**对厂商**：毛利率扩大

### 机制 3：调度折价

**现象**：单 token 边际成本通过 4 个技术手段持续下降：

| 技术 | 单 token 成本下降 |
|---|---|
| 缓存（Cache） | 命中缓存 → 降 90% |
| 量化（Quantization） | FP16 → INT8 / INT4 → 降 50-75% |
| MoE 架构 | 激活参数压到 3-5% → 断崖下降 |
| 投机解码 | 推理时间降 30-50% |

**数据**（开源领军团队 R 公开技术报告）：
- 单 token 推理成本 2024 Q4：$0.0014
- 单 token 推理成本 2026 Q1：$0.0002（**降 86%**）
- API 售价 2024 Q4：$0.27/百万
- API 售价 2026 Q1：$0.014/百万（**降 95%**）
- **卖价跌得比成本快 → 毛利率反而扩**

### 机制 4：预算内化

**现象**：AI 调用从"研发项目支出"变成"运营常规支出"。

- 2024 年：CFO 视角 = "这是研发部的新玩具"
- 2026 年：CFO 视角 = "这是水电费 + SaaS 订阅 + 云资源"

**对厂商的含义**：
- 客户内部审批路径缩短（IT 部门决定权增大）
- 续约率提升（60-80% 行业基准）
- ARPU 提升（CFO 主动加预算上限）

**数据**：
- 2024 年企业 AI 预算中位数：$50K / 年
- 2026 年企业 AI 预算中位数：$500K / 年（**涨 10 倍**）
- 主要驱动：从"试点"→"规模化"→"固定支出"三级跳

### 机制 5：合规溢价

**现象**：可审计 / 来源记录 / 私有化部署的 token 仍能溢价 30-100%。

- 金融行业：审计 token 价格 **1.3-2.0x** 标准价
- 政企客户：私有化部署 token 价格 **1.5-3.0x** 标准价
- 医疗 / 法律：可溯源 token 价格 **1.2-1.5x** 标准价

**对厂商的含义**：
- 标准 API 价格战激烈（毛利率 30-50%）
- 合规 / 私有化 / 行业定制 token 仍是高毛利（毛利率 60-80%）
- **MaaS 销售的核心利润来源**就是合规溢价，不是标准 API

### 中国特色的 3 个叠加

| 中国特色 | 节省 / 加成 | 机制 |
|---|---|---|
| **国产 GPU 替代** | 推理成本降 30-50% | 国产 A 卡价格仅为 H100 的 50-70%，TDP 接近 |
| **电力补贴** | 推理成本降 15-25% | 内蒙 / 贵州 / 宁夏 IDC 电费 0.3-0.4 元/kWh，比沿海低 30-50% |
| **开源生态飞轮** | ARR 摊薄国内固定成本 | 开源领军团队 R / 模型乙 拿下 OpenRouter 50%+ 份额 → 海外调用回流 → 国内算力闲置时段填满 → 国内 ASP 降但总毛利涨 |

**净效果**：
- 中国推理成本 = 海外 10-15%（结构）
- 中国 API 售价 = 海外 10-15%（跟随）
- **单 token 毛利率与海外持平甚至更高**（结构成本低 + 合规溢价强）

## 六、反方观点 / 风险点

### 6.1 价格 1/10 还能运转的 3 个隐忧

**隐忧 1：补贴战能持续多久？**
- 当前中国 API 价格 = 成本 + 微利（不是真正的市场出清价）
- 如果某天行业出清（剩 2-3 家寡头），价格可能反弹 5-10 倍
- **类比**：中国云市场 2018-2020 价格战 → 2021-2022 涨价 30-50%

**隐忧 2：开源飞轮能不能持续？**
- 开源领军团队 R / 模型乙 当前是"用爱发电 + 国家战略支持"
- 如果政策支持减弱，或者团队商业化压力增大 → 开源协议可能收紧
- **类比**：Red Hat / MongoDB / Elastic 都走过"开源 → 商业化收紧"的路径

**隐忧 3：合规溢价的可持续性**
- 当前合规 / 私有化溢价的 30-100% 是基于"国产替代 + 数据安全"政策
- 如果海外模型重新获准进入中国 → 合规溢价会被压缩

### 6.2 闭环的 3 个脆弱点

**脆弱点 1：单点模型的窗口期**
- 模型甲当前短剧渗透率 95% → 行业饱和
- 海外视频模型 3 代跟进 → 字节护城河能持续多久？
- **判断**：视频模型窗口期 18-24 个月，需持续投入下一代模型

**脆弱点 2：Coding 模型的同质化**
- 模型丙当前 Coding 头部，但海外模型 + Cursor / Devin 都在抢
- 海外模型一旦重新获准，Coding 头部位置可能轮换
- **判断**：Coding 窗口期 12-18 个月，需绑定开发者生态

**脆弱点 3：通用 LLM 的价格战泥潭**
- 国产开源 7B / 70B 价格已经低到 0.5-2 元/百万 tokens
- 继续降价空间有限（已经接近边际成本）
- **判断**：通用 LLM 赛道 2 年内会剩 3-5 家寡头，价格战结束

## 七、结论：商业闭环在什么条件下成立

### 7.1 成立的 4 个条件（同时满足）

✅ **条件 1**：模型在某个垂直场景做到头部（前 3 名）
✅ **条件 2**：客户业务深度绑定（迁移成本 > 切换收益）
✅ **条件 3**：Token 收入 > 算力 + 研发 + 销售成本（毛利率 > 30%）
✅ **条件 4**：有"自家鱼塘"或"开源飞轮"或"合规溢价"三选一

### 7.2 商业模式对标：海外 vs 中国

| 维度 | 海外（OpenAI / Anthropic） | 中国（某头部视频云 A / Coding 头部 X / 综合云 B） |
|---|---|---|
| **核心场景** | 开发者 + 个人 Pro | 集团内场景 + 行业 KA |
| **闭环路径** | 模型能力 + 工具链 + 生态 | 集团养鱼 + 开源飞轮 + 合规溢价 |
| **护城河** | 模型质量 + 工具链 | 自家场景 + 数据合规 + 国产化 |
| **毛利率** | 38-50% | 70-93%（头部）/ 30-50%（通用） |
| **价格战态度** | 谨慎，避免破坏全球定价 | 激进，国内先打穿再说 |
| **可持续性** | 高（CFO 接受度强） | 中（依赖政策 + 开源生态） |

### 7.3 给读者的判断

**对中国 MaaS 玩家**：
- 通用 LLM 赛道 18 个月内会出清
- 垂直场景（视频 / Coding / 短剧 / 行业 Agent）是 2026-2028 真正赚钱的主战场
- "自家鱼塘"是终极护城河，集团内场景养鱼 → Token 收割的路径值得全盘学

**对客户 / 投资人**：
- "提价不掉量"在头部 AI 模型上成立（Coding 头部 X 已经验证）
- 1/10 价格可持续 2-3 年，之后会反弹 5-10 倍（行业出清）
- 开源飞轮 = 全球调用摊薄国内成本，**开源领军团队 R / 模型乙 是结构性赢家**

**对海外读者**：
- 中国 MaaS 销售范式是"形可学神学不到"
- 真正的护城河是集团内场景养鱼——这是中国互联网大厂的独有资产
- 海外可以学"年框 + Token 包 + 效果分成"定价模型，但学不到"自家鱼塘"
