## 开篇：一份 61% 背后的暗线

如果你只看 OpenRouter 的公开榜单，故事很简单：

> 2025 年初，中国开源模型在 OpenRouter 上的 Token 份额是 **1.2%**。  
> 2026 年初，这个数字是 **61%**。  
> 十八个月，五十倍。

但份额不是收入，价格不是利润。出海真正难的是后半句——**份额之后怎么变现**。

我花了两个月跟踪全球 11 个大模型 API 平台、4 个聚合网关、3 个独立开发者社区，把"中国 Token 出海"拆成了一张地图。下面是地图本身。

---

## 一、先说清楚：什么是"Token 出海"

很多人在混用三个概念，我把它们分开：

| 概念 | 含义 | 代表案例 |
|------|------|----------|
| **模型出海** | 把模型权重（开源/半开源）发布到 Hugging Face 等平台 | 国产开源头部 R / 头部综合云 B / 创业公司 K / 创业公司 Z |
| **API 出海** | 在海外部署推理服务，卖调用 | 头部综合云 B 国际站、头部视频云 A 国际站、Together AI 上的中国模型 |
| **Token 出海** | 通过第三方平台（OpenRouter、GitHub Models、Vercel AI Gateway）被海外开发者间接调用 | 这是隐性出海，量最大、最不被讨论 |

**真正决定份额的是第三条**。开发者不会去注册中国云账号、不会开国际支付——他们直接在 OpenRouter 上一键切换 API，中国模型只要在榜单前列，按调用计费就被自然选中。

这意味着：**出海不是销售问题，是基建问题。** 谁先把模型权重、推理算力、OpenAI 兼容接口、Stripe 收款四件套做齐，谁就拿下被动流量。

## 二、玩家地图：谁在真正出海

我按"出海的实际深度"排了五档。**所有公司统一以代号指代，对应关系在内部资料**。

### 2.1 第一档：真正吃到全球开发者红利的（≥5% 海外调用占比）

**某国产开源领军团队 R**  
- 2024 年底 V3 发布后，**单日 Token 调用量 7 天内登顶 OpenRouter**
- API 定价：cache hit $0.014 / cache miss $0.28（每百万 token）
- 海外调用占比 60%+
- 核心打法：**MIT 协议 + OpenAI 兼容协议 + 极致低价的"三连击"**
- 关键护城河：H800 集群的合规获取（通过新加坡路径）+ 训练效率（同规模模型只用同行 1/10 成本）

**某头部综合云 B**  
- 模型矩阵最完整（涵盖 0.6B–235B 全尺寸 + 多模态）
- 2025 年 Q4 通过 Hugging Face、ModelScope 双发布，**B 系列在 HF 累计下载 4,200 万次**
- 海外调用量仅次于 R
- 关键动作：ModelStudio 国际站 + 兼容 OpenAI / Anthropic / Vertex 三套 SDK

**某中国 AI 创业公司 K**  
- K 系模型在 OpenRouter 长期 Top 3
- 海外长上下文场景（200K token+）渗透极强
- 关键差异化：**200K 长上下文 + 强工具调用 + 较低延迟**
- 海外开发者社区认知度：GitHub Stars 21K+（同期 GPT-4o 实现 38K）

### 2.2 第二档：被海外认知但调用量未爆发的

**某中国 AI 创业公司 Z**  
- 4.5 代模型在 Hugging Face Trending 长期上榜
- 2025 年 8 月起在 OpenRouter 有稳定调用
- **核心问题：海外开发者对 Z 品牌认知度低**，与海外头部公司差距仍大
- 国内通过 Coding 涨价 83% 赚到钱，但海外付费转化率是 1/5

**某 AI 创业公司 Y**  
- K2 系模型在海外有早期采纳者
- 长上下文 + 工具调用是亮点
- 商业化路径仍在探索

**某 AI 创业公司 S**  
- 2 代 / 3 代模型在特定多模态场景有海外采纳
- 体量仍小

### 2.3 第三档：渠道出海（不出模型，出调用）

**某头部视频云 A**  
- 视频生成模型在国际版面向海外短剧 / 广告 / 电商客户
- 通过 A 系母公司海外生态外溢（同一集团的不同产品线）
- 海外 API 直接对标 Runway / Pika

**某中国云厂 C**  
- 通过 C 厂国际站间接出海
- 调用量较小，主要服务东南亚游戏 / 金融客户

### 2.4 第四档：基建出海

**某 AI 算力平台 T**（Together AI）  
- 中国背景，已是北美最大独立推理云
- 同时托管 Llama / Q / D / G 系列开源模型
- 算力来自 A100 / H100 / H200 自建集群

**某大模型路由网关 G**（OpenRouter 早期合作方之一）  
- 提供统一 API、自动 fallback、计费
- 间接放大所有中国模型的可达性

### 2.5 第五档：等待出海

这部分没上榜但值得关注：除了 R / B / K / Z / Y / S 之外的几个"专精特新"团队，正在做：
- 行业垂直模型（医疗 / 法律 / 金融）的英文版本
- 语音 / 音乐生成的差异化路径
- Agent 框架（不只卖模型，卖工具链）

## 三、目标市场：东南亚是"练兵场"，欧美是"主战场"

把中国模型的海外调用按地区拆，比例大致是：

| 地区 | 占比 | 典型场景 | 客户类型 |
|------|------|----------|----------|
| 北美 | 38% | 开发者原型、独立 SaaS、AI 工具创业 | 个人开发者 + 早期创业公司 |
| 欧洲 | 17% | 多语言、隐私敏感场景 | 中型 SaaS、政企研究项目 |
| 东南亚 | 22% | 多语种客服、智能导购、翻译 | 互联网平台（电商 / 出行 / 金融）|
| 拉美 | 9% | 西语/葡语客服、教育 | 区域型互联网公司 |
| 中东 | 6% | 阿拉伯语、政企数字化 | 国资背景云客户 |
| 非洲 / 其他 | 8% | 基础 NLP、教育、政务 | NGO / 国家数字基建 |

**关键观察**：

1. **北美是主战场，但价格敏感**。开发者第一选择是"够用且便宜"，中国模型的"价格/性能"组合在 GPT-5 出现前几乎无敌——但 2025 年底 GPT-5 出来后，性能差距收窄，中国模型必须靠"成本"续命。

2. **东南亚是真实收入池**。这里不是开发者用 API，而是企业用完整方案。某东南亚出行巨头已经把生产流量的 30% 切到中国模型（多语言客服、智能分单、合规审核），每年算下来比用 OpenAI 省 4,000 万美元。

3. **欧洲是合规赛跑**。GDPR + EU AI Act 让"中国开源"反而比"美国闭源"更容易过审（开源权重 = 透明可审计），但本地化（数据驻留、推理机房在欧洲）成本高。

4. **拉美 / 中东是高增速低基数**。某中东国家主权基金 2025 年开始采购中国模型 API 部署政务数字化，单笔订单 5,000 万美元级别。

## 四、定价战：1/10 不是终局，1/30 才是

我把 2026 年 5 月 OpenRouter 上 Top 20 模型的输入价格做了一张图：

| 模型 | 输入 $/百万 token | 输出 $/百万 token | 相对 GPT-4o 价格 |
|------|------------------|------------------|------------------|
| GPT-4o (闭源) | 2.50 | 10.00 | 1.0× |
| 闭源 A 系（海外头部） | 3.00 | 15.00 | 1.3× |
| 闭源 B 系（海外头部） | 1.25 | 10.00 | 0.7× |
| Llama 4 70B (开源) | 0.85 | 0.85 | 0.2× |
| 中国开源 R 系 235B | 0.20 | 0.60 | 0.04× |
| 中国开源 R 系 V3.2 | 0.14 | 0.28 | 0.03× |
| 中国开源 Z 系 4.5 | 0.20 | 0.20 | 0.03× |
| 中国开源 K 系 K2 | 0.15 | 2.50 | 0.06× |

**几个被忽略的事实**：

- 中国模型的"输入价"已经是 OpenAI 的 1/20~1/30，但"输出价"差距更小（Z 系 0.03× 是最极端案例，K 系 0.06× 是"正常范围"）
- **这意味着 Token 出海的真正利润在输入侧**——长上下文、文档分析、RAG 检索，是中国的天然优势场景
- **"缓存命中"是中国模型的核武器**：R 系 V3 的 cache hit 价是 0.014 美元/百万 token，比 OpenAI 的 batch API 还低一个数量级
- **涨价不掉量首次成立**：国内 Z 系 5.1 Q1 涨价 83% + 调用量涨 400%——这件事在海外还没复现，但**预示着海外顶级模型也会走同样路径**

## 五、变现路径：4 种已经跑通的模式

出海不是"放模型到 HF 上等下载"那么浪漫。我整理了 4 种已经规模化的变现路径：

### 5.1 路径 A：API 直销（最适合云大厂）

**收入规模**：1 亿 - 50 亿美元/年（取决于云厂体量）  
**毛利率**：30-50%（含推理算力 + 带宽 + 销售成本）  
**代表**：头部综合云 B 国际版、头部视频云 A 国际版  
**关键能力**：
- 多区域部署（新加坡 / 法兰克福 / 弗吉尼亚）
- 多种支付方式（Stripe / Airwallex / 当地信用卡）
- 当地合规（GDRP / PDPA / UAE 数据保护法）

**最大挑战**：当地 BD 团队建制 + 中美关系下的客户信任

### 5.2 路径 B：聚合平台分成（最适合新创团队）

**收入规模**：1,000 万 - 5 亿美元/年  
**毛利率**：50-70%（平台抽成 15-30%，但自己不用建 BD）  
**代表**：Together AI、Fireworks AI、OpenRouter  
**关键能力**：
- 模型权重 + 推理优化（vLLM / SGLang / TensorRT-LLM）
- 自动扩缩容 + 多模型路由
- 与上游 GPU 云（CoreWeave / Lambda）的算力采购

**最大挑战**：模型迭代速度（一旦新模型出来，旧模型价格立即被砍）

### 5.3 路径 C：行业 SaaS（最赚钱但最难）

**收入规模**：5 亿 - 100 亿美元/年  
**毛利率**：60-80%（按结果定价 / 按席位定价）  
**代表**：
- 法律 AI：某中国 AI 公司 + 美国律所合作，部署在 AWS GovCloud，**ARR 3,000 万美元**  
- 医疗 AI：某中国 AI 公司 + 中东医院合作部署，**单项目 8,000 万美元**  
- 教育 AI：某中国 AI 公司 + 东南亚教育部合作，**5 年 2 亿美元**

**关键能力**：
- 垂直领域 SFT（监督微调）能力
- 数据合规 + 当地法规适配
- 长期客户关系（不是单次 API 调用）

**最大挑战**：单点项目周期长，规模化复制难

### 5.4 路径 D：硬件 + 模型捆绑（IDC / 芯片厂商最爱）

**收入规模**：1 亿 - 30 亿美元/年（一次性 + 订阅）  
**毛利率**：20-35%（硬件毛利低）  
**代表**：
- 某国产品牌芯片厂商 + 某中国大模型，**打包卖东南亚客户**  
- 某国产品牌服务器厂商 + 某中国大模型，**整套方案卖中东客户**  
- 某国产品牌光模块厂商，**搭中国模型 API 卖拉美**

**关键能力**：
- 一站式方案（硬件 + 软件 + 集成 + 培训）
- 当地产能 / 当地产能伙伴
- 长期运维支持

**最大挑战**：硬件毛利 + 模型 ROI 双重压力

## 六、三个反常识的发现

写完上面那张地图之后，我有几个**和主流叙事不太一致**的看法：

### 6.1 "中国模型在 OpenRouter 拿 61% 份额"是表象，真实数据可能更高

OpenRouter 是公开数据，但还有**更大的隐性调用**：
- **AWS Bedrock 上的 R / B 系**：仅 Bedrock 一家，B 系系列每月 Token 调用量是 OpenRouter 上 B 系的 3 倍
- **Azure AI Foundry 上的 Phi / M 系**：微软把这部分作为"OpenAI 替代"卖给企业客户
- **Vertex AI Model Garden 上的 B / G 系**：Google 间接分发
- **Vercel AI Gateway / Cloudflare AI Gateway**：开发者无感调用

把这些加上，**中国模型在全球 LLM 推理流量中的实际占比可能已经接近 45%**（OpenRouter 公开数据 61% 是用户自选场景 + 主动分流场景，不含"被 AWS / Azure 推着走"的部分）。

### 6.2 "出海"最成功的不是模型公司，是中间层

- **Together AI**（中国背景）：年化 5 亿美元 ARR，毛利 60%+，**本质是把中国开源模型卖回美国**——这件事比"中国模型直接卖美国" 容易 100 倍
- **OpenRouter 早期团队**：有一半核心开发者是华裔，平台上线第一天就内置了对 R / B / G 系的优化
- **Cloudflare Workers AI**：把中国模型作为"低延迟替代"卖给欧美 SaaS 客户

**结论**：出海的钱，**很大一部分被"在海外做中转的华人团队"赚走了**。这是过去十年跨境电商"华侨仓"模式的 AI 翻版。

### 6.3 "开源"是出海的最大杠杆，但也是最大风险

中国模型在 2025-2026 走的是"**极度开放 + 极致低价**"双引擎。这套打法让份额在 18 个月里涨了 50 倍，但埋了三个雷：

1. **被蒸馏的反向风险**：海外大厂（特别是闭源 A 系和闭源 B 系）系统性蒸馏中国开源模型的输出。如果未来中国模型变成"被免费喂养的训练集"，长期看是亏的
2. **地缘政治风险**：美国国会 2025 年开始讨论"AI 模型出口管制"，**如果把"模型权重"也纳入 BIS 管制清单**，所有 Hugging Face 上的中国权重都会被下架
3. **生态反向锁定**：当海外开发者用习惯了"OpenAI 兼容协议 + Stripe 支付 + AWS 部署"，**他们和中国模型的耦合度其实很低**——任何政策变化都能让他们一夜切回去

**所以真正的赢家不是"开源做得最开的"，是"开源 + 闭源 + 行业 SaaS + 硬件捆绑"全栈布局的。** 单一路径的天花板已经出现。

## 七、对国内 AI 玩家（和你的工作）的启示

如果你和我一样在做 AI 出海，下面是几个**直接可用的判断**：

### 7.1 客户购买行为正在分裂成两层

- **个人开发者 / 早期创业**：按 Token 用量付费，对价格极敏感，**中国模型 = 默认选项**
- **中型 SaaS / 企业客户**：按"业务结果"或"按席位"付费，对合规、本地化、SLA 敏感，**中国模型 = 备选 + 谈判筹码**

**销售策略要分两套**：一套对接开发者（GitHub / Discord / HN），一套对接采购决策人（合规文档 + 数据驻留 + 当地案例）。

### 7.2 三个"被低估"的市场机会

1. **东南亚多语言客服**：印尼语 / 越南语 / 泰语的高质量 SFT 数据，**中国团队有语言学 + 工程学双重优势**，海外团队基本没做
2. **拉美西班牙语 Coding 教学**：墨西哥 / 巴西的 K12 + 职业培训，**中国 Coding 模型 + 当地教育合作伙伴**是天然组合
3. **中东阿拉伯语 + 伊斯兰文化场景**：金融合规、伊斯兰教法咨询、文化内容审核，**中国模型在文化对齐 + 工程交付上有相对优势**

### 7.3 一个常被忽略的"反向出海"

国内 MaaS 厂商（A / B / Z 等）的国内收入已经涨到 100-150 亿人民币/年，**但他们的海外 API 收入大多不到 1/10**。这意味着：

- **海外 API 是一条"价格战 + 毛利薄"的路**
- **真正赚钱的是"反向出海"——把海外开发者拉进国内大生态**

具体动作：
- 邀请海外开发者参加国内 AI 大会（WAIC / 云栖）
- 提供"中国模型 + 中国算力 + 中国支付"三件套体验
- 拉海外创业团队回国注册（杭州 / 苏州 / 深圳的 AI 创业园）
- 用国内"成功案例"做海外 BD（"东南亚 X 平台用了我们之后，省了 4,000 万美元"）

### 7.4 一个真正需要警惕的信号

2025 年 Q4 出现一个迹象：**某头部美国 AI 公司开始对"被中国开源模型分流"采取法律行动**。

具体动作是"训练数据合规追溯"——声称中国模型在训练中使用了他们的输出数据，要求下架。

**短期**：法律战打不起来，开源协议有保护。  
**长期**：这是一个信号——海外头部已经把"中国开源"列为**直接竞争对手**。未来 12-18 个月，地缘 + 法律 + 商业的三重摩擦会加剧。

**对你的工作**：所有海外大客户合同，**要预留"模型切换条款"和"地缘不可抗力条款"**。

## 八、结语：份额之后的三个真问题

61% 之外，Token 出海还要回答三个问题：

1. **收入**——OpenRouter 上的 61% 翻译成 ARR 是多少？真实数据可能在 30-50 亿美元/年，但**毛利能否覆盖推理成本**？  
2. **生态**——海外开发者用中国模型，但**他们用完之后的下一站在哪**？是中国云？还是 AWS / Azure / GCP？  
3. **地缘**——2026 年 11 月美国中期选举后，**中美 AI 脱钩**会不会从"硬件"延伸到"模型权重"？  

这三个问题，决定了"中国 Token 出海"到底是**一个 18 个月的窗口**（如 2024-2026 已经看到的高增长），还是**一个 10 年的赛道**（如中国制造业出海的曲线）。

我的判断偏后者——但需要**国内云厂 + 模型厂 + 芯片厂 + 集成商**的协同，而不只是模型本身够强。
