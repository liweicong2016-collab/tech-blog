## 1. 执行摘要：三条主线决定 2026 年 MaaS 胜负

![三条主线：Token 规模化 / 成本曲线下探 / 生态入口重排](/tech-blog/images/maas-ppt/01-three-main-lines.svg)

**主线一：Token 规模化** —— 日均调用量成为云厂商 AI 商业化第一指标。字节披露豆包日均 Token 使用量超 120 万亿，阿里披露百炼 Token 消耗三个月提升 6 倍。

**主线二：成本曲线下探** —— 价格从按量 API 走向输入/输出 Token、上下文长度、Batch、缓存、套餐包等多层结构，厂商用低价模型锁定开发者。

**主线三：生态入口重排** —— 阿里偏云与企业全栈，字节偏应用流量与内容/Agent，腾讯偏企业连接、微信生态与 TokenHub 聚合入口。

> **投资与业务含义**：MaaS 的核心不是单模型毛利，而是"高频 Token 流量 + 低边际推理成本 + 应用生态转化"的组合收益。

来源：火山引擎、阿里云百炼、腾讯云 TokenHub、公开报道

## 2. Token 经济：收入设计与推理成本耦合

![Token 经济公式：Token = usage + pricing + compute](/tech-blog/images/maas-ppt/02-token-economics.svg)

### 收入端

- 按输入/输出 Token 分别计价，**输出通常更贵**
- 长上下文、思考模式、多模态、视频生成形成价格梯度
- Token Plan、包年包月和企业承诺消费把波动调用转成确定收入

### 成本端

- GPU/国产 AI 芯片、HBM、网络与电力决定推理底层成本
- Batch 调用、上下文缓存、蒸馏、小模型路由压低单位成本
- Agent 长链路会放大 Token 消耗，也提高粘性与单客价值

### 关键变量 × 商业含义 × 管理重点

| 关键变量 | 商业含义 | 管理重点 |
|---|---|---|
| **每百万 Token 单价** | 决定开发者门槛与竞品替换成本 | 价格带、套餐、免费额度 |
| **单位推理成本** | 决定 MaaS 毛利弹性 | 模型压缩、调度、缓存、芯片供给 |
| **Token 留存率** | 决定是否从试用转为生产环境 | SLA、工具链、数据安全、Agent 框架 |

> **一句话公式**：`Token = usage + pricing + compute`

<a id="maas-market"></a>
## 3. MaaS 市场：从模型 API 升级为 AI 操作系统

![MaaS 三层堆栈：模型层 / 平台层 / 应用层](/tech-blog/images/maas-ppt/03-maas-stack.svg)

| 层级 | 能力 | 价值捕获 | 竞争焦点 |
|---|---|---|---|
| **模型层** | 通用、推理、代码、多模态、视频、语音 | 按 Token/任务收费 | 性能、成本、上下文、开源/闭源策略 |
| **平台层** | 模型市场、API 网关、评测、监控、数据治理 | 企业合同、套餐、云资源联动 | 开发体验、SLA、合规与私有化 |
| **应用层** | Agent、办公、客服、营销、搜索、代码生成 | 订阅、增值、广告、电商转化 | 场景闭环、用户入口、工作流粘性 |

> **2026 关键判断**：竞争不再是"谁有模型"，而是谁能把模型嵌入企业流程与消费者入口，并把高频调用沉淀成可预测的 Token 收入。

<a id="bytedance"></a>
## 4. 字节火山引擎：用应用流量拉 Token 飞轮

![字节火山引擎：豆包 120 万亿 + 入口矩阵](/tech-blog/images/maas-ppt/04-bytedance.svg)

**核心数字：豆包日均 Token 使用量超 120 万亿**

公开报道显示，豆包大模型日均 Token 使用量已超过 120 万亿，体现从 C 端应用到企业 API 的调用规模优势。

### 产品打法

豆包 2.0 系列覆盖 **Pro、Lite、Mini、Code** 等梯度，以不同能力/成本匹配 Agent、编程和通用场景。

### 商业化抓手

- **火山方舟**承接企业 API
- **豆包 / 即梦 / 抖音**生态贡献应用场景
- **TRAE** 等开发工具放大代码类 Token 消耗

### SWOT

| 优势 | 约束 |
|---|---|
| C 端高频入口、内容生成场景丰富、价格激进 | 企业云心智弱于阿里/腾讯，需证明 SLA 与行业交付 |

> **观察指标**：企业 MaaS 收入、Token Plan 留存、Agent 应用转化

来源：新浪科技；火山引擎开发者社区，豆包 2.0 API

## 5. 阿里百炼：让 AI 成为云的增长层

![阿里百炼：3 月 6 倍 + 价格梯度](/tech-blog/images/maas-ppt/05-alibaba.svg)

**核心数字：百炼 Token 消耗三个月提升 6 倍**

阿里管理层披露，百炼 MaaS 平台公共模型服务市场 Token 消耗三个月提升 6 倍。

### 价格结构

百炼官方价格页显示，Qwen 系列按 **上下文长度、输入/输出、思考模式** 分层；Batch 调用与上下文缓存提供折扣。

### 全栈叙事

从 **AI Infra、芯片/服务器、模型、百炼平台到千问应用官网**，阿里试图把 Token 收入与云资源消耗绑定。

### SWOT

| 优势 | 约束 |
|---|---|
| 云客户基础、企业交付、开源 Qwen 生态、价格体系透明 | C 端超级入口相对分散，低价竞争压缩短期毛利 |

> **观察指标**：百炼收入占阿里云比例、企业 Agent 项目转生产率

来源：阿里云百炼官方价格页；新浪财经；每经网

<a id="tencent"></a>
## 6. 腾讯混元 + TokenHub：聚合入口押注

![腾讯 TokenHub：50-80% 折扣 + 聚合入口](/tech-blog/images/maas-ppt/06-tencent.svg)

### 平台入口

**腾讯云 TokenHub 定位为统一大模型服务入口**，整合混元并引入第三方模型，覆盖对话、推理、代码、视觉、图像和视频等场景。

**套餐折扣：50%-80%** —— TokenHub 产品页强调套餐比 API 按量计费便宜 50%-80%，用预付费和用量包降低开发者边际成本感知。

### 模型迭代

混元 Hy3 preview 已开源发布，公开资料显示输入价约 **1.2 元/百万 Tokens**，并面向 Agent 开发者推出 Token Plan。

### SWOT

| 优势 | 约束 |
|---|---|
| 微信/企微/腾讯会议/云与安全生态，聚合平台利于企业多模型治理 | 需要在自研模型心智与第三方聚合之间平衡价值捕获 |

> **观察指标**：TokenHub 付费包复购、混元开源生态、微信场景调用量

来源：腾讯云 TokenHub 产品页；腾讯云开发者社区

## 7. 竞争矩阵：三家从不同需求控制点切入

![三家对比矩阵：6 维度热力图](/tech-blog/images/maas-ppt/07-matrix.svg)

| 维度 | 字节 | 阿里 | 腾讯 |
|---|---|---|---|
| **核心入口** | 豆包、抖音、即梦、TRAE、火山方舟 | 阿里云百炼、千问、企业云客户 | 腾讯云 TokenHub、混元、微信/企微生态 |
| **Token 增长来源** | C 端高频交互、内容生成、编程 Agent | 企业应用、云上 AI 工作流、开源生态迁移 | 企业协作、微信生态、第三方模型聚合 |
| **定价风格** | 高性价比模型 + 低门槛计划 | 分层透明价格 + Batch/缓存折扣 | 套餐锁价 + 聚合调用优惠 |
| **最大机会** | 把 C 端流量训练成企业级 MaaS 收入 | 让 MaaS 成为阿里云最大产品线之一 | 成为企业多模型调用与治理入口 |
| **主要风险** | 企业交付与合规心智建设 | 价格战与算力成本对毛利的压力 | 聚合平台稀释自研模型差异化 |

> 战略对比基于公开产品定位、价格页和 2026 年公开报道

<a id="implications"></a>
## 8. 战略含义：重塑云、软件与开发者产品设计

![影响图：云 / 企业 / 开发者 / 投资人 四象限](/tech-blog/images/maas-ppt/08-implications.svg)

### 对云厂商

- MaaS 将成为云资源的新前台，背后绑定 GPU、存储、网络和数据安全
- 价格战不可避免，关键是用缓存、Batch、模型路由和自研芯片维持成本优势

### 对企业客户

- 采购标准从模型榜单分数转向 **总拥有成本、稳定性、数据隔离和工作流集成**
- Agent 项目要建立 Token 预算、效果归因和自动降级策略

### 对开发者

- **多模型路由、提示词缓存、上下文压缩、结果复用**会成为应用利润率基本功

### 对投资者

- 应跟踪 **Token 消耗、付费留存、单位推理成本、AI 云收入占比**，而不是只看模型发布节奏

<a id="outlook"></a>
## 9. 2026-2027 展望：从低价普惠到效率分层

![2026-2027 outlook：趋势时间轴](/tech-blog/images/maas-ppt/09-outlook.svg)

### 三项预测

1. **价格继续分层**：旗舰模型涨价或稳价，Lite/Mini 模型承担普惠流量
2. **企业 MaaS 从试点转生产**：Agent、客服、代码、营销内容先放量
3. **平台从卖 API 转向卖治理**：模型市场、审计、路由、成本控制成为标配

### 建议

| 角色 | 建议 |
|---|---|
| **厂商** | 用 Token Plan 锁定高频开发者，同时公开 SLA 与成本优化能力 |
| **企业** | 建立多供应商策略，按任务拆分模型层级，避免单一模型成本锁定 |
| **开发者** | 把 Token 成本纳入产品设计，默认做缓存、压缩和模型路由 |
